# 文献整理助手

将用户提供的论文或文献任务整理成可复用、可持续追加的结构化文档。默认先做逐篇抽取，再做跨文献综合，避免停留在泛泛摘要。目标不是只整理信息，而是尽量生成接近真实论文综述的分析结果。对方法类论文，额外重建“从原始数据到最终输出”的完整技术链路。

## 概述

本 skill 面向中文用户，核心能力限定为“读 + 比 + 整理”。默认产物是一个统一的 Markdown 文档，包含逐篇文献卡片、跨文献比较与综合、以及参考文献清单。

仅当用户明确要求“帮我找相关论文”“搜索某方向文献”“补充这个领域的代表性工作”时，才进入可选检索流程。否则应优先处理用户已经提供的论文、链接、笔记或现有文档。

如果用户目标是形成综述草稿，而不是简单资料整理，则应在输出中主动加入：研究问题界定、方法流派划分、证据质量判断、研究脉络重建、争议与研究空白，而不是停留在逐篇概述层面。

## 适用任务

- 阅读单篇论文，并输出结构化卡片，而不是泛泛摘要
- 比较多篇论文，统一抽取字段并整理成一个对比文档
- 将用户已提供的多篇文献信息汇总成综述草稿、研究笔记或课程作业素材
- 在既有文档中继续追加新的论文卡片、比较结果和综合段落
- 用户明确要求时，先检索某主题论文，再按同一模板纳入整理文档
- 对方法类论文，重建从 raw data 到模型输入、数据转换、方法类型、最终产出的完整方法学链路

## 主流程

严格按以下五步执行，不要跳过“逐篇抽取”直接写结论。

### 1. 识别任务类型

先判断当前任务属于哪一类：

- 单篇阅读
- 多篇比较
- 多篇整理成综述草稿
- 明确要求时的可选检索

如果任务核心是“阅读、比较、整理已给文献”，不要自行扩展成检索任务。

### 2. 确认输入材料与目标产物

在深入阅读前，先明确：

- 输入材料是什么：PDF、网页链接、题录、现有笔记、已有 Markdown 文档
- 用户真正想要什么：单篇卡片、多篇比较、综述草稿、研究资料积累
- 最终是否要写入一个持续维护的统一文档

默认目标产物是一个统一 Markdown 文档；若用户已有文档，则优先在原文档结构基础上追加，而不是分散写成多个零碎文件。

### 3. 按统一字段逐篇抽取信息

对每篇论文使用统一抽取字段。字段定义和单篇卡片模板见 `references/extraction-schema.md`。

抽取时遵守以下规则：

- 先区分“论文明确写了什么”与“基于内容的合理推断”
- 缺失信息必须标注“文中未明确报告”，不要自行补全
- 只抽取与用户任务相关的内容，避免无关细节堆积
- 多篇论文任务中，所有论文尽量使用同一套字段，保证可比性

如果论文核心是介绍一个方法、模型或分析流程，不要只总结研究结论。额外启用“方法学链路总结”，追踪：

- 原始 raw data 是什么
- 模型或方法真正消费的数据形态是什么
- raw data 如何一步步变成模型输入
- 方法本体属于神经网络、传统机器学习、生信工具流程或混合路线
- 方法最终产出什么，以及这些产出如何被解释、评估和使用

### 4. 先做逐篇卡片，再做跨文献综合

不要把多篇论文的单篇摘要简单拼接后结束。完成逐篇卡片后，必须补一层综合分析，至少回答下列部分问题：

- 这一主题的研究脉络是什么
- 各篇论文在哪些问题上结论一致
- 各篇论文在哪些方法、数据、结论上存在分歧
- 常见局限是什么
- 当前主题的研究空白在哪里

跨文献综合的组织模板见 `references/review-template.md`。

如果用户希望结果更接近真正的论文综述，则综合部分还应补充：

- 这一领域的研究问题是如何被定义的
- 不同论文分别属于哪些方法流派
- 各流派的关键假设、优势、代价和适用前提是什么
- 现有结论的证据强度是否一致
- 当前比较是否公平，是否存在数据、标签、评估口径不一致的问题
- 研究发展是否呈现明显时间演化

如果当前任务重点是方法比较，综合部分优先回答：

- 不同论文的 raw data 起点是否一致
- 模型输入的数据表示是否不同
- 数据转换流程中哪些步骤最影响结果
- 方法本体分别属于哪类技术路线
- 最终输出是预测值、可解释特征、排序结果、曲线、图谱还是其他产物

同时注意：不同方法不是静态并列存在的，而是随时间演化的。比较方法时，不只比较当前维度差异，还要说明：

- 某一时期的主流技术路线是什么
- 后续方法相对于前一阶段改进了哪一环
- 技术演化是发生在数据表示、预处理、模型结构、解释性还是输出形式
- 某些“更强”的方法是否依赖了更新的数据条件、标签资源或计算能力

### 5. 输出到一个统一文档

默认将结果组织到一个文档中，结构包括：

- 文献清单
- 逐篇文献卡片
- 跨文献比较与综合
- 研究空白与争议
- 可直接复用的结论
- 参考文献

除非用户明确要求分文件输出，否则不要把信息散落到多个单独笔记里。

## 统一抽取规则

多篇论文任务默认使用以下字段作为标准 schema：

- 题目
- 作者
- 年份
- 来源信息
- 研究问题
- 研究对象或任务
- 数据集/语料/实验对象
- 方法
- 基线/对照
- 评价指标
- 主要结果
- 作者声称的贡献
- 局限性
- 与当前研究主题的相关性

如果用户目标是综述式写作，除结果字段外，还要显式提取：

- 研究问题定义
- 核心假设
- 证据质量
- 比较公平性风险
- 研究流派归属
- 时间位置或发展阶段

当论文属于方法类、模型类或流程类工作时，除上述通用字段外，再补充以下“方法学链路”字段：

- 原始数据类型
- 模型实际需求数据
- raw data 到模型输入的转换步骤
- 方法类型
- 任务定义
- 标签或真值来源
- 评估方式与关键指标
- 最终产出
- 可解释性
- 适用边界

如果论文之间存在明显的前后继承关系，还应额外记录：

- 相对前一代方法的改进点
- 相对前一代方法保留了哪些限制
- 该方法在时间演化中的位置

如果某项信息缺失，明确写“文中未明确报告”。不要用猜测填补字段。

单篇卡片的默认模板、字段解释、以及字段缺失时的写法，统一参考 `references/extraction-schema.md`。

## 方法学链路总结

对方法类论文，不只复述作者结论，而要重建“从原始数据到最终输出”的完整技术链路。

重点回答以下问题：

- 作者最初拿到的 raw data 是什么
- 模型或方法真正使用的数据表示是什么
- raw data 经过了哪些 preprocessing、bioinformatics pipeline、特征提取或编码步骤
- 方法本体是生信工具流程、传统机器学习、深度学习/神经网络，还是混合方法
- 最终输出是什么，以及这些输出如何被解释、评估和使用

建议同时补充以下关键信息：

- 任务定义是分类、回归、排序、检测还是注释
- 标签或真值来自药敏实验、数据库注释、人工标注还是临床结局
- 评估指标是什么，是否有外部验证或独立测试
- 方法是否具有可解释性，可解释对象是什么
- 适用于哪些样本、病原体、测序条件或研究边界

如果任务包含多篇方法论文比较，还要把每篇方法放回时间轴上理解：

- 它出现于该领域的哪个阶段
- 它是在替代旧路线、补强旧路线，还是把旧路线与新模型结合
- 它的优势是否来自真正的方法创新，还是来自更丰富的数据、更新的标签、更宽松的评估条件

方法学链路字段的详细模板见 `references/extraction-schema.md`，默认综述中如何组织这些信息见 `references/review-template.md`。

## 输出文档规则

默认输出为一个可持续追加的 Markdown 文档，结构如下：

```markdown
# 主题

## 任务范围

## 文献清单

## 逐篇文献卡片

## 跨文献比较与综合

## 研究空白与争议

## 可直接复用的结论

## 参考文献
```

具体写作要求：

- 逐篇卡片在前，跨文献综合在后
- 对比性信息优先用表格或统一小标题保持结构一致
- 综合部分不要按论文顺序罗列，而应按主题、方法脉络、争议点或研究空白组织
- 若用户已有文档框架，优先复用其结构，但仍保留“逐篇卡片 + 综合”这两层

如果用户要的是“像真正综述一样”的结果，优先扩展为以下结构：

```markdown
# 主题

## 研究背景与问题定义

## 纳入文献与分类原则

## 方法流派与时间演化总览

## 逐篇文献卡片

## 跨文献比较与综合

## 证据质量与比较公平性评议

## 研究空白与争议

## 未来方向

## 参考文献
```

完整模板见 `references/review-template.md`。

## 引用与出处规则

默认使用作者-年份型引用方式，例如：

- `张三，2022`
- `(Smith, 2023)`

第一版不要求严格符合某一具体期刊的参考文献格式，但必须尽量保留：

- 题名
- 作者
- 年份
- 来源信息
- 链接或文件来源（如果有）

正文中的综合性判断应能追溯到至少一篇逐篇卡片中的信息。不要写无法定位来源的强结论。

## 可选检索流程

仅在用户明确要求“搜索某方向论文”时启用本节流程。

执行检索时：

- 先说明检索主题、关键词或筛选方向
- 先给出有限数量、可管理的候选论文，不直接堆很长清单
- 说明为什么这些论文被纳入当前整理文档
- 检索得到的论文仍然要按统一抽取字段纳入逐篇卡片

第一版不设计系统综述级别的纳入/排除标准，不覆盖严格数据库检索协议。

## 验证清单

完成输出前检查：

1. 是否已经区分单篇卡片与跨文献综合，而不是只有摘要堆叠
2. 是否所有论文都使用了统一字段，且关键信息没有串错
3. 是否缺失信息被明确标注，而不是被推测补全
4. 是否综合段落回答了研究脉络、共同结论、分歧、局限或研究空白中的至少几项
5. 是否最终结果被整理进一个统一文档
6. 是否正文默认采用作者-年份式引用，并在文末保留参考文献清单
7. 如果任务是综述式写作，是否分析了研究问题、方法流派、证据质量和研究空白
8. 如果任务涉及多代方法，是否体现了时间演化，而不是把所有方法当作静态并列对象
