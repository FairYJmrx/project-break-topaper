# 默认综述文档模板

默认输出为一个可持续追加的 Markdown 文档。先写逐篇卡片，再写跨文献综合。

## 默认结构

```markdown
# 主题

## 任务范围

说明当前整理的主题、研究问题、使用场景，以及本次纳入文献的范围。

## 文献清单

列出当前纳入的论文标题、作者、年份和来源信息。

## 方法流派与时间演化总览

先概括这一领域经历了哪些主要路线，以及这些路线如何随时间演化。

## 逐篇文献卡片

### 文献卡片：论文 A

- 作者：
- 年份：
- 来源信息：
- 研究问题：
- 研究对象或任务：
- 数据集/语料/实验对象：
- 方法：
- 基线/对照：
- 评价指标：
- 主要结果：
- 作者声称的贡献：
- 局限性：
- 与当前研究主题的相关性：
- 研究问题定义：
- 核心假设：
- 证据质量：
- 比较公平性风险：
- 研究流派归属：
- 时间位置或发展阶段：

### 文献卡片：论文 B

...

## 跨文献比较与综合

从方法、数据、结论、争议点或时间脉络中选择合适维度进行综合。

如果当前主题主要是方法比较，优先从以下维度组织：

- raw data 起点
- 模型输入的数据表示
- raw data 到模型输入的转换路径
- 方法类型
- 最终产出及用途
- 研究流派
- 时间演化路径

## 证据质量与比较公平性评议

单独评价不同论文的验证强度、数据与标签可比性、评估口径一致性，以及横向比较时可能存在的不公平因素。

## 研究空白与争议

指出当前文献中尚未解决的问题、结论不一致之处、常见局限。

## 未来方向

总结这一领域下一阶段可能沿着哪些路线发展，以及哪些限制最值得被优先突破。

## 可直接复用的结论

给出可以直接用于综述、研究笔记、课程作业或项目调研的表述。

## 参考文献

保留题名、作者、年份、来源信息，正文默认使用作者-年份式引用。
```

## 综合段落必须回答的问题

跨文献综合部分至少应回答以下几类问题中的多数：

- 这一主题的研究脉络是什么
- 各篇论文在哪些问题上形成共同结论
- 各篇论文在哪些方法、数据或解释上存在分歧
- 常见局限有哪些
- 研究空白或后续值得跟进的问题是什么
- 方法路线是否随时间发生演化，以及演化的驱动力是什么

如果是方法类主题，额外优先回答：

- 不同方法依赖的数据起点是否一致
- 数据预处理和特征构造是否构成主要差异
- 不同方法最终输出的形式是否不同
- 可解释性、评估方式和适用边界分别如何
- 后续方法相对前一阶段究竟改进了什么，又继承了什么限制

## 写作规则

- 不要只把单篇卡片顺序拼接后结束。
- 综合段落要比单篇卡片更高一层，体现比较、归纳和判断。
- 结论必须能追溯到逐篇卡片中的信息。
- 若已有用户文档，可沿用原结构，但应尽量保留“逐篇卡片 + 综合综述 + 参考文献”三层。
- 对方法类论文，优先呈现“数据链路 + 方法类型 + 最终产出”的对照关系，而不是只比较最终准确率。
- 对时间跨度较大的文献组，不要把早期和晚期方法直接并列评价；先说明它们所处的发展阶段，再比较其设计选择。
