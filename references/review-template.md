# 默认综述文档模板

这个模板用于把多篇论文整理成一份可持续追加的 Markdown 文档。原则是先逐篇卡片，再做跨文献综合。

## 推荐结构

```markdown
# 主题

## 任务范围

说明当前整理的主题、研究问题、使用场景，以及这次纳入文献的边界。

## 文献清单

列出纳入的论文题目、作者、年份、来源信息。

## 方法流派与时间演化

概述这个领域经历了哪些主要阶段，以及这些阶段如何沿时间演化。

## 逐篇文献卡片

### 文献卡片：论文 A

- 标题：
- 作者：
- 年份：
- 发表 venue：
- 研究问题：
- 研究对象或任务：
- 数据集 / 语料 / 实验对象：
- 方法：
- 基线 / 对照：
- 评价指标：
- 主要结果：
- 作者声称的贡献：
- 局限性：
- 与当前主题的相关性：
- 时间位置 / 发展阶段：
- 对我们工作的启发：
- 是否值得重点精读：

### 文献卡片：论文 B

... 以同样字段继续

## 跨文献比较与综合

从方法、数据、评价、解释和时间演化几个维度进行比较，不要只按论文顺序罗列。

建议至少覆盖：

- 研究问题如何被定义
- 各论文的共同结论与分歧点
- 数据与标签是否公平可比
- 方法演化是由数据、表示、模型还是评估驱动
- 当前研究空白与下一步方向

## 证据质量与比较公平性评估

说明不同论文之间在数据规模、任务定义、标签来源、评估指标和实验条件上的差异，判断结论的可比性。

## 研究空白与争议

指出当前文献中尚未解决的问题、互相矛盾的结论，以及最值得继续追踪的方向。

## 可直接复用的结论

给出可以直接用于写综述、研究笔记、开题或项目决策的结论句。

## 参考文献

保留题目、作者、年份、来源信息。正文默认使用作者-年份制引文。
```

## 综合写作重点

综合段落至少要回答大部分下面的问题：

- 这个主题的研究问题如何被定义
- 不同论文分别属于什么方法流派
- 各流派的核心假设、优势和限制是什么
- 是否存在数据、标签或评价口径不一致的问题
- 研究演化是否呈现明显的时间阶段
- 当前研究空白在哪里

如果主题是方法比较，还要优先回答：

- 不同方法依赖的原始数据是否一致
- 数据预处理和特征构造是否构成主要差异
- 最终输出形式是否不同
- 可解释性、评估方式和适用边界是否不同
- 后续方法相对前一阶段到底改进了什么

## 写作规则

- 综合部分要比单篇卡片高一层，不能只是把卡片拼接
- 结论必须能追溯到至少一篇卡片中的证据
- 缺失信息不要补写成事实
- 如果用户已有文档结构，优先沿用用户结构，但保留“逐篇卡片 + 跨文献综合”两层
- 如果论文跨越较长时间，不要把早期和晚期方法直接并排做静态比较，要先说明各自处于什么发展阶段
