# 对策与文献图谱规则

用于把每个难题连接到候选方案和跨领域文献来源。

## 核心原则

文献图谱不是参考文献列表，而是“问题-对策-来源领域-论文价值”的关系图。

每条文献线都要说明：

- 它解决什么问题；
- 它来自哪个领域；
- 它支持当前方案，还是提示当前方案风险；
- 当前方案相对它的新意是什么；
- 哪些实验必须补齐，才能说服审稿人。

## 文献来源分层

### 1. 本领域基线文献

说明当前任务已有主流方法。

例：

- 传统工具
- 数据库方法
- benchmark
- 领域综述

作用：

- 定义审稿人期待的 baseline；
- 证明问题真实存在；
- 避免只和弱 baseline 比较。

### 2. 跨领域方法来源

说明当前方案的思想来自哪里。

例：

- 计算机视觉的 margin loss
- 长尾学习
- 度量学习
- 校准
- open-set recognition
- MoE

作用：

- 解释方法来源；
- 找到可迁移论文机会；
- 避免把已有思想包装成全新。

### 3. 任务特异改造文献

说明为什么不能直接套用跨领域方法，必须进行领域改造。

作用：

- 形成论文创新点；
- 明确本项目的独特困难；
- 给出实验设计依据。

## 输出模板

```markdown
## 文献图谱：难题名称

### 本领域基线

- 文献：
- 解决的问题：
- 局限：
- 对当前项目的意义：

### 跨领域方法来源

- 文献：
- 方法思想：
- 可借鉴部分：
- 不能直接套用的原因：

### 当前方案的位置

- 继承了什么：
- 改造了什么：
- 新解决了什么：
- 必须补的证据：
```

## 与其他 skill 的配合

- 用户要求查新文献时，调用或遵循 `nature-academic-search`。
- 用户要求逐篇精读时，回到本 skill 的 `extraction-schema.md`。
- 用户要求把文献写成论文段落时，调用或遵循 `nature-writing`。
